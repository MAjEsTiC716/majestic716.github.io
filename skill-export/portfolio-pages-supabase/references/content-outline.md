# Content Outline

Use this as the default narrative for resume-driven engineering portfolios with an interactive playground.

## Site map

- Home
- About
- Experience
- Projects
- Skills
- Playground
- Contact

## Positioning

- Engineer first
- Systems thinker second
- Interactive builder third

## Hero formula

- Name
- Role
- One short line about backend, automation, testing, cloud, or systems work
- Two primary actions:
  - view projects
  - open playground

## About formula

- One paragraph on professional identity
- One paragraph on how the person works or what kinds of problems they solve
- Optional short card on what the site is meant to demonstrate

## Experience formula

For each role:

- title
- company
- location
- dates
- 3 to 5 bullets that emphasize shipped work, systems impact, and technical breadth

## Projects formula

For each featured project:

- project title
- short summary
- tech tags
- 3 to 4 bullets on problem, implementation, constraints, and what was learned

## Playground formula

- brief intro that connects the game area to frontend skill
- local practice game
- authenticated multiplayer game
- short explanation of backend shape

## Contact formula

- email
- phone when appropriate
- GitHub
- resume PDF
